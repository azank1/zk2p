📊 Matching Algorithm Explained

Price-Time Priority (Pro-Rata with FIFO)

The matching engine uses a two-stage sorting algorithm:

orders.sort_by(|a, b| a.price.cmp(&b.price).then(a.created_at.cmp(&b.created_at)))

Stage 1: Sort by price (ascending) - cheapest ask orders matched first
Stage 2: Sort by timestamp (ascending) - older orders matched first (FIFO)
Matching: Bid matches all asks where ask.price ≤ bid.price

Key Features
  • ✅ Partial fills supported (order can be split across multiple matches)
  • ✅ Orders stored in on-chain PDA (OrderBook account)
  • ✅ Tokens held in escrow until settlement
  • ✅ Cross-program invocation (CPI) to OrderStore for matched orders
